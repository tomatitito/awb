# Mobile-first responsive workspace for AWB

Coordinate the responsive redesign of AWB across phone, tablet/small laptop, and desktop widths. Establish viewport-aware layout composition, implement a true mobile single-focus workspace, and preserve desktop multi-pane productivity workflows.

Initial scope should cover layout architecture, mobile navigation, mobile Kanban, mobile Graph, and mobile Details/Agent behavior.

## Acceptance Criteria

- an implementation plan exists for desktop, tablet, and mobile compositions
- viewport-aware layout switching is implemented or tracked through child tickets
- phone-sized screens use a single-focus workspace instead of compressed desktop panes
- mobile Kanban, Graph, Details, and Agent experiences are tracked through child tickets
- desktop multi-pane workflows remain available
- bun run build succeeds for delivered work
