# Normalize YAML date values when parsing ticket metadata

parseTicket only keeps created when gray-matter returns a string. YAML date values are commonly parsed as Date objects, so unquoted created dates can disappear from the Details metadata.

## Acceptance Criteria

created metadata accepts string and Date-like YAML values. Date values are serialized consistently in the API response. Invalid created values are handled predictably. Parser tests cover quoted and unquoted dates.
